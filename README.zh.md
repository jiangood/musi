# Literal Musi

听。搜。继续。

基于 Git 同步的极简本地音乐播放器。

## 为什么？

还记得你只是拥有音乐文件的时候吗？

没有流媒体。没有推荐。没有算法告诉你该喜欢什么。

只有你选择的歌曲文件夹。

音乐服务已经变得嘈杂——算法随意编排的歌单、你不想要的推荐、版权到期就消失的歌曲。

你的音乐应该属于你。

## 核心理念

Literal Musi 建立在一个简单的原则上：

**把音乐放进去。听。想找什么就搜。不需要就删掉。**

没有歌单。没有评分。没有算法推荐。没有云端。

只有文件。

## 功能特点

- **听**：简洁的播放器，带基本控制
- **搜**：按文件名搜索所有音乐
- **同步**：Git 同步（GitHub），让音乐库纳入版本管理
- **极简**：没有数据库，没有元数据抓取，没有专辑封面下载

## 工作原理

音乐存储在简单的目录结构中：

```
repo/
├── pile/    ← 音乐文件
└── trash/   ← 已删除文件（仅 Git 同步，用于恢复）
```

没有数据库。没有音乐库 XML。只有文件。

## 为什么没有歌单？

因为你不会去翻歌单。

你听专辑。你搜索那首特定的歌。你随机播放一个文件夹。

歌单不过是保存下来的搜索查询，假装自己是什么了不起的东西。

如果你需要歌单，建个文件夹就好了。

## 为什么没有元数据？

ID3 标签、专辑封面、按流派分类——这些都需要数据库、扫描器和持续的维护。

这个应用把音乐当作文件对待。文件名就是标题，文件夹就是专辑。

如果你需要丰富的元数据，请使用其他播放器。

## 适合谁？

这个应用可能适合这样的人：

- 拥有自己的音乐文件
- 希望用版本管理来管理音乐库
- 用搜索代替浏览
- 不需要推荐告诉你该听什么
- 喜欢简单、可预测的工具
- 熟悉基于 Git 的工作流

## 哲学

Literal Musi 不是关于整理你的音乐库。

而是关于**不需要整理**。

这个应用故意保持简单。如果一个功能需要分好几步来解释，那它可能不属于这里。

## 使用方法

### 添加音乐

1. 将音乐文件复制到 `pile/` 目录
2. 运行 `git add` 和 `git push`（或在应用中同步）
3. 你的音乐在所有连接的设备上都可以使用

### 多设备同步

Git 同步让你的音乐在多个设备之间保持同步：

- 启动应用时自动同步
- 不要在多台设备上同时修改同一个文件
- 如果发生冲突，先同步的设备胜出
- 所有变更都会保留在 Git 历史中
